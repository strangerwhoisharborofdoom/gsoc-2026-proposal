# Contributing

Thanks for your interest in contributing to this GSoC proposal and related repositories. Small, focused contributions and early feedback from maintainers are very welcome.

## How to contribute

1. Fork the repo and create a branch named `fix/short-description` or `feat/your-feature`.
2. Make your changes and write clear commit messages.
3. Run tests locally (if applicable) and ensure lints pass.
4. Open a Pull Request describing the change and link related issues.

## Coding standards
- Python: follow Black and type hints where practical.
- C++: follow Google C++ style where applicable.

## Reporting issues
Open an issue with a clear title and reproduction steps. Tag it appropriately: `bug`, `enhancement`, `question`.

## Community
Be respectful. Follow the Code of Conduct.
